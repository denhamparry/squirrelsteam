## Approach: bitmap → true vector (trace, not embed)

The logo is a clean two-tone mark — an ideal tracing candidate. We trace it into
real SVG paths with `potrace` (run via `nix run nixpkgs#potrace`), **not** embed
the raster inside an SVG wrapper. Pipeline (documented in this plan so it is
reproducible):

```bash
SRC="examples/Rhiwbina 1415/rhiwbina-rfc-logo-with-ears.jpg"
# Mark only: crop the wordmark off (top 552px). For the full lockup, drop the
# -crop. -shave removes the source's 3px edge hairline so it is not traced as a
# stray full-width line; -negate flips white-on-black to black-on-white;
# -threshold makes it bilevel; -trim + even -border give a tight, centred canvas.
magick "$SRC" -gravity North -crop 740x552+0+0 +repage -shave 3x3 \
  -colorspace Gray -resize 2400x -negate -threshold 50% \
  -trim +repage -bordercolor white -border 60 -type bilevel /tmp/mark.pbm
nix run nixpkgs#potrace -- /tmp/mark.pbm -b svg -t 8 -a 1.0 -O 0.3 --flat -o /tmp/mark.svg
npx --yes svgo --multipass /tmp/mark.svg -o <dest>   # minify + clean
# Post-process: set fill="currentColor", drop width/height (keep viewBox),
# add role="img" + aria-label.
```

POC verified: the traced result is crisp (smooth squirrel curves, legible
lettering). `-shave` removes the source edge hairline that earlier traced as a
stray top line.

## Deliverables

### Master vector assets (`src/assets/logo/`)

1. **`squirrel-mark.svg`** — squirrel symbol only (text cropped off),
   transparent background, `fill="currentColor"` so it adopts the surrounding
   text colour. A clean `viewBox` (no `pt` units / DOCTYPE / metadata after
   svgo).
2. **`rhiwbina-rfc-logo.svg`** — full lockup (squirrel + "RHIWBINA R.F.C."),
   transparent background, `fill="currentColor"`.

`currentColor` keeps both masters monochrome-flexible: white in the dark header,
black on paper, etc.

### Public assets (stable URLs — `public/`)

3. **`favicon.svg`** — replace the placeholder "S". Dark rounded square
   (`#1a1a1a`, `rx` matching current) with the **white squirrel mark** centred.
   Mark-only because the wordmark is illegible at favicon sizes.
4. **`og-image.png`** — 1200×630, white full logo centred on `#1a1a1a`. Raster
   (not SVG) because social scrapers don't reliably render SVG. Generated by
   rasterising the full-logo SVG (`resvg`) onto a black canvas with `magick`.

### Wiring

5. **`src/components/Header.astro`** — replace the 🐿️ emoji `.brand__mark` with
   the inline `squirrel-mark.svg` (inherits the header's white via
   `currentColor`). Preserve `aria-hidden`/labelling; constrain size in CSS.
6. **`src/layouts/BaseLayout.astro`** — add `og:image`, `og:image:width`,
   `og:image:height`, `og:image:alt`, and `twitter:image` (absolute URLs built
   from `Astro.site`). This makes the existing `summary_large_image` Twitter
   card and OG tags actually render an image.

## Decisions

- **Trace, don't embed** — a real vector mark scales for header/favicon/print.
- **`fill="currentColor"`** on masters — one asset serves light and dark
  contexts.
- **Mark-only favicon** — wordmark unreadable at 16–32px.
- **Raster OG image** — SVG OG images are unreliable across social platforms.
- **Zero-padded branch** `gh-issue-009` to match repo convention (e.g. `#15`).
